## 场景

检查当前仓库剩余未提交内容，并将确认无误的内容按规范提交。

### 开发目标

#### 1. 开发内容

- 检查当前 git 状态，定位剩余未提交目录。
- 校验 `new-file/` 技能目录及引用资源是否完整。
- 为本次任务补齐 `req`、`res`、`todo`、`tests`、`logs` 记录并完成提交。

#### 2. 预期结果

- 明确当前剩余内容仅包含应提交的 `new-file/` 目录。
- `new-file/` 技能和资源文件通过基本结构检查。
- 本次检查与提交过程具备可追踪的测试、日志和 git 提交记录。

#### 3. 验证方式

- 检查 `git status --short` 确认剩余变更范围。
- 检查 `new-file/SKILL.md` 及其引用资源文件存在且无诊断问题。
- 提交后读取最新一次提交信息，确认标题概括任务且正文列出核心改动点。

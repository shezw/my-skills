## 场景

为 React 项目新增一套独立、可自动生效的高质量工程规范，默认采用成熟 UI 栈并强调可读性与干净模块拆分。

### 开发目标

#### 1. 开发内容

- 创建独立的 `react-standards` 技能目录和 `SKILL.md`。
- 编写 React 规范正文、UI 默认栈说明、评审清单和目录结构示例。
- 创建用户级 `react.instructions.md`，对 React 组件文件默认生效。
- 为本次任务补齐 `req`、`res`、`todo`、`tests`、`logs` 记录并完成提交。

#### 2. 预期结果

- 形成一套独立的 React 项目规范技能。
- 规范明确默认 UI 栈为 Mantine + `@tabler/icons-react`，并覆盖模块化、可读性、测试、可访问性和性能要求。
- 本次任务完成后存在可追踪的测试、日志与 git 提交记录。

#### 3. 验证方式

- 检查 `react-standards/` 目录结构、`SKILL.md` 和引用资源是否完整。
- 检查用户级 `react.instructions.md` 是否存在且 `applyTo` 覆盖 React 组件文件。
- 搜索规范文本，确认包含 Mantine、Tabler、模块拆分、hooks、测试、a11y 和性能等关键条目。
- 提交后读取最新一次提交信息，确认标题概括任务且正文列出核心改动点。

## 场景

新增 Rust 语言规范，覆盖模块化、可读性、注释以及补充的关键工程约束。

### 开发目标

#### 1. 开发内容

- 新增用户级 Rust 指令文件，对 `*.rs` 文件默认生效。
- 新增仓库内镜像文档，沉淀同一套 Rust 规范。
- 为本次任务补齐 `req`、`res`、`tests`、`logs` 记录。

#### 2. 预期结果

- 形成一份可直接生效的 Rust 编码规范。
- 规范明确覆盖模块化深度、文件规模、注释要求与其他关键工程规则。
- 本次任务完成后存在可追踪的测试、日志与提交记录。

#### 3. 验证方式

- 检查新增规范文件 frontmatter 与结构是否完整。
- 搜索规范文本，确认覆盖模块化、注释、错误处理、测试、unsafe、lint 等关键条目。
- 检查 git 最新提交，确认已按默认提交规则提交本次变更。

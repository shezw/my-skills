## 场景

调整 Rust 规范的技能目录结构，使其成为独立、可发现的 Rust 项目编写标准，并暴露到基线技能入口。

### 开发目标

#### 1. 开发内容

- 创建独立的 `rust-standards` 技能目录和 `SKILL.md`。
- 迁移 Rust 规范文档到该技能下，并补充必备的检查清单。
- 在 `agent-baseline/SKILL.md` 和工作流中显式暴露 Rust 语言规范入口。
- 为旧路径保留兼容转向页，并补齐本次任务的 `req`、`res`、`todo`、`tests`、`logs` 记录。

#### 2. 预期结果

- Rust 规范以独立技能形式存在，路径和职责清晰。
- `agent-baseline` 能明确指向 Rust 语言专项规范。
- 仓库内包含 Rust 规范正文、检查清单和兼容转向说明。

#### 3. 验证方式

- 检查 `rust-standards/` 目录结构、`SKILL.md` 和引用资源是否完整。
- 检查 `agent-baseline/SKILL.md` 与工作流是否显式暴露 Rust 技能入口。
- 提交后读取最新一次提交信息，确认标题概括任务且正文列出核心改动点。

## 场景

为 Go 项目新增一套独立、可自动生效的工程规范，兼顾单体与微服务模式、清洁架构和成熟工程治理。

### 开发目标

#### 1. 开发内容

- 创建独立的 `go-standards` 技能目录和 `SKILL.md`。
- 编写 Go 规范正文、架构默认项、评审清单和目录结构示例。
- 创建用户级 `go.instructions.md`，对 Go 文件默认生效。
- 为本次任务补齐 `req`、`res`、`todo`、`tests`、`logs` 记录并完成提交。

#### 2. 预期结果

- 形成一套独立的 Go 项目规范技能。
- 规范明确覆盖清洁架构、接口隔离、配置驱动、统一网关、多协议、错误体系、事件优先、AI-Ready、可观测性与安全底线。
- 本次任务完成后存在可追踪的测试、日志与 git 提交记录。

#### 3. 验证方式

- 检查 `go-standards/` 目录结构、`SKILL.md` 和引用资源是否完整。
- 检查用户级 `go.instructions.md` 是否存在且 `applyTo` 覆盖 Go 文件。
- 搜索规范文本，确认包含清洁架构、Wire、config、REST/gRPC/WebSocket、错误码、事件总线、OpenTelemetry、安全和 AI-Ready 关键条目。
- 提交后读取最新一次提交信息，确认标题概括任务且正文列出核心改动点。

## 场景

调整通用基线中的提交策略，使编码任务默认提交，并要求提交信息体现核心改动点。

### 开发目标

#### 1. 开发内容

- 更新 `agent-baseline` 技能和参考文档中的提交规则为默认提交。
- 更新用户级常驻基线指令中的提交规则为默认提交。
- 增加提交信息模板，并在日志模板中记录提交标题与核心改动点。

#### 2. 预期结果

- 规范文本明确要求开发结束后默认提交。
- 提交信息要求标题概括任务、正文列出核心改动点。
- 本次修改完成后执行实际提交，并能通过 git 记录验证。

#### 3. 验证方式

- 搜索规范文本，确认不再保留“仅在明确要求时提交”的旧规则。
- 检查提交模板和日志模板内容是否完整。
- 执行 git 提交，并检查最新一次提交信息是否包含核心改动点。
